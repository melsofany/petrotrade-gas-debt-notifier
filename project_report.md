# تقرير دمج مشروع WhatsApp Sender Dashboard

## 1. المقدمة

بناءً على طلبكم، تم دمج الكود المقدم لمشروع "WhatsApp Sender Dashboard" في بيئة عمل محلية. يهدف هذا المشروع إلى إرسال رسائل واتساب من جداول بيانات جوجل (Google Sheets) مع توفير لوحة تحكم (Dashboard) لإدارة العملية وعرض الإحصائيات.

تمت معالجة الملفات `package.json` و `server.js` التي قدمتموها، بالإضافة إلى إنشاء ملف `.env` لتكوين المتغيرات البيئية اللازمة.

## 2. تفاصيل المشروع المدمج

### 2.1. `package.json`

تم إنشاء ملف `package.json` بالبيانات التي قدمتموها، والذي يحدد معلومات المشروع الأساسية والتبعيات (dependencies) المطلوبة. يتضمن الملف التبعيات التالية:

| التبعية           | الإصدار   | الوصف                                      |
| :---------------- | :-------- | :----------------------------------------- |
| `dotenv`          | `^16.0.3` | لتحميل المتغيرات البيئية من ملف `.env`    |
| `express`         | `^4.18.2` | إطار عمل (framework) لتطبيقات الويب       |
| `express-session` | `^1.17.3` | لإدارة جلسات المستخدمين                    |
| `googleapis`      | `^118.0.0`| للوصول إلى واجهات برمجة تطبيقات جوجل (Google APIs)، وتحديداً Google Sheets |
| `qrcode-terminal` | `^0.12.0` | لعرض رمز الاستجابة السريعة (QR Code) في الطرفية |
| `whatsapp-web.js` | `^1.21.0` | مكتبة للتحكم في واتساب عبر الويب           |

### 2.2. `server.js`

يحتوي ملف `server.js` على منطق الخادم الرئيسي، والذي يشمل:

*   **إعداد الخادم**: باستخدام `express` لإدارة المسارات (routes) والوسيطات (middleware).
*   **المصادقة (Authentication)**: نظام تسجيل دخول بسيط للوحة التحكم باستخدام كلمة مرور محددة في المتغيرات البيئية.
*   **تكامل Google Sheets**: وظائف لقراءة أرقام الهواتف من جدول بيانات جوجل وتحديث حالة الإرسال وتسجيل الجلسات.
*   **تكامل WhatsApp**: تهيئة عميل واتساب باستخدام `whatsapp-web.js`، بما في ذلك التعامل مع رمز الاستجابة السريعة (QR Code) وحالة الاتصال.
*   **مسارات API**: توفير نقاط نهاية (endpoints) للوحة التحكم لإرسال الرسائل، والتحقق من حالة واتساب، وعرض الإحصائيات.
*   **قالب رسالة واتساب**: يتضمن الكود قالب رسالة باللغة العربية لإرسال إنذار قانوني نهائي بخصوص مديونية الغاز الطبيعي.

### 2.3. `.env` (المتغيرات البيئية)

تم إنشاء ملف `.env` في جذر المشروع لتخزين المتغيرات البيئية الحساسة. يجب عليكم تعديل هذا الملف بالقيم الصحيحة قبل تشغيل المشروع. المتغيرات المطلوبة هي:

*   `PORT`: المنفذ الذي سيعمل عليه الخادم (القيمة الافتراضية: `3000`).
*   `GOOGLE_SERVICE_ACCOUNT_BASE64`: مفتاح حساب الخدمة الخاص بجوجل مشفرًا بصيغة Base64. يستخدم هذا للمصادقة مع Google Sheets API.
*   `SPREADSHEET_ID`: معرف جدول بيانات جوجل الذي يحتوي على أرقام الهواتف.
*   `WHATSAPP_SESSION_BASE64`: (اختياري) بيانات جلسة واتساب مشفرة بصيغة Base64 لاستعادة الجلسة السابقة وتجنب مسح رمز الاستجابة السريعة في كل مرة. سيتم إنشاء هذا بعد أول مصادقة ناجحة.
*   `DASHBOARD_PASSWORD`: كلمة المرور المطلوبة للوصول إلى لوحة التحكم.

## 3. تعليمات التشغيل

لتشغيل المشروع، يرجى اتباع الخطوات التالية:

1.  **الانتقال إلى دليل المشروع**: 
    ```bash
    cd /home/ubuntu/petrotrade-gas-debt-notifier
    ```

2.  **تعديل ملف `.env`**: افتح ملف `.env` باستخدام محرر نصوص وقم بتعبئة القيم الصحيحة للمتغيرات البيئية المذكورة أعلاه. على سبيل المثال:
    ```
    PORT=3000
    GOOGLE_SERVICE_ACCOUNT_BASE64="YOUR_BASE64_ENCODED_SERVICE_ACCOUNT_KEY"
    SPREADSHEET_ID="YOUR_GOOGLE_SHEET_ID"
    WHATSAPP_SESSION_BASE64=""
    DASHBOARD_PASSWORD="your_secure_password"
    ```
    *ملاحظة*: للحصول على `GOOGLE_SERVICE_ACCOUNT_BASE64`، يجب إنشاء حساب خدمة في Google Cloud Platform، وتنزيل مفتاح JSON الخاص به، ثم تشفير محتويات ملف JSON إلى Base64. يمكن القيام بذلك باستخدام أمر مثل `cat your-service-account-key.json | base64` على نظام Linux/macOS أو أدوات عبر الإنترنت.

3.  **تثبيت التبعيات**: تم تثبيت التبعيات بالفعل، ولكن إذا كنت تقوم بتشغيل المشروع في بيئة جديدة، يمكنك تثبيتها باستخدام:
    ```bash
    pnpm install
    ```

4.  **بدء تشغيل الخادم**: 
    ```bash
    pnpm start
    ```
    أو
    ```bash
    node server.js
    ```

5.  **الوصول إلى لوحة التحكم**: بعد بدء تشغيل الخادم، يمكنك الوصول إلى لوحة التحكم عبر متصفح الويب على العنوان `http://localhost:3000/login` (أو المنفذ الذي حددته في ملف `.env`).

6.  **مصادقة واتساب**: عند الوصول إلى لوحة التحكم لأول مرة، قد تحتاج إلى مسح رمز الاستجابة السريعة (QR Code) الذي سيظهر في الطرفية (console) أو على لوحة التحكم نفسها باستخدام هاتفك المتصل بواتساب. بعد المصادقة بنجاح، سيتم حفظ جلسة واتساب تلقائيًا (وسيتم طباعة `WHATSAPP_SESSION_BASE64` في الطرفية لتمكينكم من حفظها كمتغير بيئي لاستخدامها لاحقًا).

## 4. ملاحظات وتوصيات

*   **عدم العثور على مستودع GitHub**: لم يتم العثور على مستودع GitHub `melsofany/petrotrade-gas-debt-notifier` من خلال عمليات البحث. هذا يعني أن الكود المقدم هو المصدر الوحيد للمشروع. إذا كان هناك مستودع عام، يرجى توفير الرابط الصحيح.
*   **الأمان**: كلمة المرور للوحة التحكم والمتغيرات البيئية الأخرى حساسة. تأكد من استخدام كلمات مرور قوية وعدم مشاركة ملف `.env` في أنظمة التحكم في الإصدار (مثل Git).
*   **استمرارية جلسة واتساب**: الكود يتضمن آلية لمحاولة حفظ واستعادة جلسة واتساب. بعد المصادقة الأولى، سيتم طباعة `WHATSAPP_SESSION_BASE64` في الطرفية. يوصى بحفظ هذه القيمة كمتغير بيئي في ملف `.env` لضمان عدم الحاجة إلى مسح رمز الاستجابة السريعة في كل مرة يتم فيها إعادة تشغيل الخادم.
*   **تنسيق أرقام الهواتف**: الكود يقوم بتنسيق أرقام الهواتف لإضافة رمز الدولة (20 لمصر) وإضافة `@c.us` ليتوافق مع متطلبات `whatsapp-web.js`. تأكد من أن أرقام الهواتف في جدول بيانات جوجل يمكن تنسيقها بشكل صحيح.
*   **معدل الإرسال**: يوجد تأخير لمدة 3 ثوانٍ بين كل رسالة واتساب يتم إرسالها (`await new Promise(resolve => setTimeout(resolve, 3000));`) لتجنب حظر الحساب. يمكن تعديل هذا التأخير حسب الحاجة، ولكن يجب توخي الحذر.
